Testing in browser

To run the Jasmine tests in your browser, go to the following pages:

  • http://localhost/handsontable/test/jasmine/SpecRunner.html - (Handsontable main test suite)
  • http://localhost/handsontable/src/3rdparty/walkontable/test/jasmine/SpecRunner.html - (Walkontable test suite)

Testing with Grunt (CLI)

To run Jasmine tests in your command line (using PhantomJS), first install Grunt (based on the instructions provided on page Building).

Useful commands:

  • grunt test - runs all test cases (Handsotnable and Walkontable)
  • grunt test:handsontable - runs only Handsontable tests
  • grunt test:walkontable - runs only Walkontable tests
  • grunt test --filter autocomplete - runs only tests and suites matching the name "autocomplete"

Testing with Sauce Labs

  1. Create a Sauce account
  2. Save your account credentials in env variables SAUCE_USERNAME and SAUCE_ACCESS_KEY (instructions: general, Mac)
  3. In Gruntfile.js, browsers is the array of browsers that will be tested. You can adjust the browser definition list to test only the browsers you want (makes it faster).
  4. In Gruntfile.js, concurrency should have the same value as your Sauce account's "Parallel tests" limit (3 for open source projects)
  5. Execute grunt sauce (or grunt sauce:handsontable, grunt sauce:walkontable)
  6. Log in to your Sauce account to see the results

More details:
