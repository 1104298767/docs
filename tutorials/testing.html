Testing in browser

To run the Jasmine tests in your browser, go to the following pages:

  • http://localhost/handsontablePro/test/UnitRunner.html - Runs the unit tests in the browser.
  • http://localhost/handsontablePro/test/E2ERunner.html - Runs the end-to-end tests in the browser. Note, that it runs only the Pro test suite, unless npm run test:handsontable was run before (in that case, the Handsontable Free tests will be processed), as running the tests in the CLI generates the E2ERunner template.

Testing with NPM (CLI)

To run Jasmine tests in your command line (using PhantomJS), first install all necessary dependencies by executing npm install. After the NPM finishes fetching required modules your workspace is ready to test.

Useful commands:

  • npm test - runs all test cases (Handsotnable and Handsontable PRO).
  • npm test:e2e - runs only Handsontable PRO end-to-end tests.
  • npm test:unit - runs only Handsontable PRO unit tests.
  • npm test:handsontable - runs only Handsontable CE end-to-end tests.
  • npm run test:e2e --testPathPattern=selection - runs only end-to-end tests and suites matching the filename "selection".
  • npm run test:unit --testPathPattern=array - runs only end-to-end tests and suites matching the filename "array".
  • npm run test:e2e.dump - generates the test/E2ERunner.html file which can by executed in the browser environment (Chrome, PhantomJS etc.) to check if tests passes.
  • npm run test:unit.dump - generates the test/UnitRunner.html file which can by executed in the browser environment (Chrome, PhantomJS etc.) to check if tests passes.
  • npm run test:e2e.dump -- --watch - generates the E2ERunner.html file on every change detected in test files. It can be helpful for debugging proposes.
  • npm run test:unit.dump -- --watch - generates the UnitRunner.html file on every change detected in test files. It can be helpful for debugging proposes.
