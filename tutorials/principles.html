We follow these simple rules which enable us to focus on performance instead of coming up with ideas that kill it or distract our attention in any other way.

  1. Performance is our number one priority (we use this Performance Lab to perform the tests)
  2. The code must be tested and bug-free
  3. The code must be well-documented
  4. The code must be easy to extend
  5. We improve the code quality through frequent patch releases
  6. We test each release against recent versions of the most popular browsers
