1.0.0

Released on 20th of January, 2016

  • Backward incompatible change
    The afterValidate hook will not provide the *visual* row indexes, instead of the *logical* ones. For example, if the first row becomes the seventh row after sorting the table, the afterValidate hook will provide 6 as the row index. To translate the *visual* row to the *logical* row, you can use the translateRow method of the columnSorting plugin. 
    var logicalRow = hotInstance.getPlugin('columnSorting').translateRow(row);
    Related issue: Handosntable Open Source issue 3132
  • Fix problem, where pasting a dataset which row count exceeds the number of rows in the table nothing happened. (18)
  • Fixed problems with dropdown menu and submenu positioning. (5)
  • Fixed problems with filtering the fixed rows. (4)
  • Fixed problem with filter options missing after filtering out all rows. (1)
